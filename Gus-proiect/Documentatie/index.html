Tehnologii folosite : Tehnologii folosite:

1.HTML
Denumirea vine de la HyperText Markup Language si se defineste ca fiind un limbaj de formatare si structurare a documentelor web utilizand o varietate de etichete si atribute.


HTML ofera:
-portabilitate:fiecare browser il suporta
-simplu si usor de inteles
-poate incorpora alte limbaje


2.CSS

Cascading Style Sheets (CSS) este un limbaj style sheet folosit pentru a descrie aspectul si formatarea unui document scris intr-un limbaj markup. Aplicarea sa cea mai comuna este in paginile web scrise in HTML si XHTML, insa poate fi aplicat la orice tip de document XML.
CSS ofera:
-vizualizarea continutului adaptat pe toate device-urile
-prin separarea continutului de design,se reduce timpul de transfer a fisierelor.


3. JavaScript

JavaScript este un limbaj de programare creat de Netscape, serveste la dezvoltarea aplicatiilor web si de cele mai multe ori ruleaza ca o aplicatie client-side care ruleaza intr-un browser web.
JavaScript ofera:
-interactivitate prin punerea la dispozitie de elemente pe care HTML nu le utilizeaza
-este un limbaj client-side, functiile JavaScript si calculele se executa repede, deoarece informatiile nu trebuie sa navigheze pe Internet.


4.PHP si MYSQL

PHP este un limbaj de programare web server-side, ceea ce inseamna ca ruleaza pe un server, permitand dezvoltatorilor web sa creeze site-uri dinamice si aplicatii web-based software ce interactioneaza cu utilizatorii si baze de date.
Pentru stocarea datelor in baze de date folosim MySQL care este cel mai des utilizat server de baze de date open source. Se caracterizeaza prin flexibilitate, stabilitate, fiabilitate si viteza in prelucrarea datelor stocate. MySQL foloseste Structured Query Language (SQL), limbajul cel mai frecvent folosit pentru adaugarea, accesarea si prelucrarea datelor in bazele de date.


5.Ajax si Jquery

AJAX se defineste ca fiind o grupare de tehnici de dezvoltare web care accesorizeaza limbajul
JavaScript si care permite paginilor web sa fie mai interactive si sa se comporte ca aplicatii locale (client-side). Altfel spus, AJAX permite unei pagini web preluarea de date de pe server fara a reincarca toata pagina.
JQuery este o librarie JavaScript opensource, utilizata de dezvoltatorii web pentru a efectua interactiuni Ajax in paginile web.

Vulnerabilitati : Vulnerabilitati :

Vom incerca sa fixam urmatoarele tipuri de vulnerabilitati:
1) Remote File Inclusion ( RFI )
2) Local File Inclusion ( LFI )
3) Cross Site Scripting ( XSS )
4) HTTP Header Injection
5) SQL Injection
6) Login ByPass
7) Remote Code and Command Execution
8) Full Path Disclosure

1) Remote File Inclusion
- consta in includerea unui fisier aflat pe un alt server folosind un parametru GET ;
- scriptul va include direct fisierul specificat prin valoarea unei variabile trimise prin GET;
- cum fixam vulnerabilitatea : schimbarea ideii incluziunii - vom folosi un switch, care va contine o lista cu paginile care pot fi incluse .

2) Local File Inclusion
- principiul este acelasi ca si la RFI ;
- scriptul va verifica daca acel fisier exista , iar daca exista in va include ;
- se pot include fisiere de pe server ;
- cum fixam vulnerabilitatetea : la fel ca si in cazul RFI . De asemenea , se pot filtra sirurile "../" sau"..\".

3) Cross Site Scripting
- o vulnerabilitate de tip client-side care nu afacteaza serverul insa va afecta clientul ;
- intalnita la nivelul scriptului ce se foloseste de cookie-uri ;
- ideea de baza : scriptul va citit o variabila prin GET , apoi afiseaza valoarea ei in browser (JavaScript) . Se testeaza variabilele prin GET carora li se atribuie o valoare cu cod JavScript ;
- codul JavaScript va fi interpretat in browserul victimei ;
- la cookie se ajunge foarte usor prin "document.cookie" in JS , si de cele mai multe ori se foloseste un cookie grabber(script PHP care primeste prin GET cookie-urile victimelelor si le scrie intr-un fisier ) pentru furtul acestora .
- cookie-urile se pot trimite catre grabber prin JavaScript foarte usor;
- se poate folosi un iframe iar pagina va putea contine orice ;
- cum fixam vulnerabilitatetea : PHP ne ofera 2 htmlentities si htmlspecialchars (transforma caracterele speciale in entitati HTML) astfel nu se va mai executa niciun cod JS .
- chiar daca s-a reusit furtul de se poate opri aceasta actiune prin : crearea unei noi variabile de sesiune in care se va memora adresa IP in momentul logarii . Cand utilizatorul va accesa o pagina se va verifica IP , daca nu coincide se va efectua delogarea .

4) HTTP Header Injection

- nu toate headerele care provin de la utilizator sunt corecte, nici User-Agent, nici Cookie ;
- se poate folosi pentru SQL Injection, daca de exemplu se cauta in baza de date Ã®n functie de un cookie;

5) SQL Injection

- un atacator poate afala informatii despre baza de date datorita unor interogari facute peste acea baza de date ;
- cum putem fixa vulnerabilitatea : se foloseste mysql_real_escape_string(adauga back-slashuri in fata unor caractere speciale);
- !! aceste caractere au o semnificatie speciala ;

6) Login ByPass
- atac intalnit in casutele de logare ;
- cum fixam vulnerabilitatetea : verificam daca exista userul , apoi verificam daca este corecta parola si apoi facem logarea ;
- ( in baza de date parola va fi salvata criptat) .

7) Remote Code and Command Execution

- consta in executarea unor comenzi PHP;
- lucru posibil datorita functiei "eval" cand se evalueaza un parametru ;
- daca utilizatorul va face request catre:

http://site.com/script.php?text=Un text';phpinfo();

- va observa in browser rezultatul functiei phpinfo();

- consta in executarea unor comenzi in Terminal ( cmd );

8) Full Path Disclosure

- e poate descoperi calea completa pentru un fisier;
- cum fixam vulnerabilitatetea : folosind functia "is_array".
